The process

Three steps, zero effort

Configure once, receive daily. Specola runs silently in your menubar and handles everything — from fetching to analysis to formatting.

01 — FETCH

Concurrent RSS harvest

Specola parses your OPML file and fetches all feeds in parallel using a 20-thread pool. Failed feeds are skipped gracefully — one broken source never blocks the rest.

feedparser handles every RSS/Atom variant.
Configurable time window: 6–72h.
02 — ANALYZE

Claude reads everything

The full digest is piped to the Claude Code CLI together with your professional profile. Claude prioritizes, connects dots, and writes analytical paragraphs — not one-line summaries.

Runs via subprocess with 300s timeout.
If Claude is unavailable, raw content is
still delivered with a warning banner.
03 — DELIVER

Premium DOCX report

The analysis is rendered into a professionally typeset document with branded headers, accent colors, clickable source links, and clear visual hierarchy.

python-docx with Calibri, 1.3 line-height.
Saved to ~/Documents/Specola/.
The output

Not a summary.
An analysis.

Each briefing is a multi-page document with deep analysis, source links, and actionable insights — all tailored to your role, your stack, and your current projects.

Da sapere oggi 5–7 key developments with context, implications, and direct links to original sources
Richiede attenzione Actionable items: what happened, why it matters for your specific context, and what to do this week
Per area tematica 3–4 sentence analytical paragraphs per topic, connecting news to broader industry trends and patterns

Why Specola

Built for professionals
who value their time

Not another feed reader. Specola is a personal intelligence analyst that lives in your menubar and works while you sleep.

🛡

No API keys, no cloud

Runs entirely on your Mac using the Claude Code CLI. Your feeds, your profile, and your briefings never leave your machine. Zero billing management, zero vendor lock-in.

🎯

Tailored to your profile

Describe your role, tech stack, interests, and current projects in a freeform markdown file. Specola filters and prioritizes everything through your professional lens.

🔗

Every claim has a source

Clickable hyperlinks to every original article, directly in the document. The briefing is a starting point for deeper reading, not a dead end.

⏰

Set and forget scheduling

Pick your preferred time, toggle auto-generate. A 60-second internal timer checks conditions silently. If your Mac is asleep, the briefing runs at wake. No launchd, no cron — it just works.

📄

Premium document output

Branded headers with diamond accent, colored section borders, Calibri typography, page numbers, and footer branding. A document you'd share with your board, not a raw text dump.

🌍

Italian & English

Full briefings in either language with localized prompts and regional context awareness. Same depth, same structure, same quality. Switch anytime in Settings.

🛠

Graceful degradation

If Claude CLI becomes unavailable, Specola still generates a DOCX with your raw feed content and a clear warning banner. You always get a briefing — analysis is a bonus, not a dependency.

📦

~15 MB native app

Pure Swift and SwiftUI. No Electron, no web views. The Python engine is bundled inside the app and auto-deploys a virtual environment on first launch. Nothing to configure manually.

📂

OPML import from anywhere

Export your feeds from Feedly, NetNewsWire, Inoreader, or any reader that supports OPML. Drop the file in Specola's settings and you're done.

Under the hood

Two components, one contract

The Swift app handles UI and scheduling. The Python engine does the thinking. They communicate via CLI arguments in and JSON on stdout out — clean, testable, independently deployable.

Swift App
UI & Scheduling
MenuBarExtra — native macOS menubar integration with dynamic badge icon
Settings Window — four tabs: Feeds, Scheduling, Profile, Advanced
Scheduler — 60-second timer with wake-from-sleep detection via NSWorkspace
Notifications — native macOS alerts when your briefing is ready
Login Item — SMAppService integration for auto-start on boot
Python Engine
Analysis & Generation
feed_fetcher.py — OPML parsing + concurrent RSS fetching with ThreadPoolExecutor
prompt_builder.py — assembles system prompt with user profile and feed digest
analyzer.py — invokes Claude Code CLI via subprocess (300s timeout)
doc_generator.py — DOCX rendering with markdown parsing and branded styling
Tests — 46 pytest tests for engine, 15 XCTest for Swift app

# Clone the repository git clone https://github.com/amargiovanni/specola.git cd specola # Build the app xcodebuild -project Specola.xcodeproj -scheme Specola -configuration Release -derivedDataPath build # Install to Applications cp -R build/Build/Products/Release/Specola.app /Applications/
FIRST LAUNCH

Auto-setup

The Python engine extracts to Application Support and creates its own virtual environment. No manual pip install needed.

CONFIGURE

Import & personalize

Drop your OPML file in the Feeds tab. Write your professional profile in the Profile tab. Pick your language and time window.

SCHEDULE

Set your time

Choose when you want your briefing (default: 07:00). Enable auto-generate and optionally add Specola to login items.

The only external dependency is claude in your PATH. Everything else — Python packages, virtual environment, output directory — is handled automatically on first launch.

Common questions

FAQ

No. Everything runs locally on your Mac. Your feeds are fetched directly, your profile stays on disk, and the analysis happens through the Claude Code CLI which runs as a local subprocess. There is no Specola server, no telemetry, and no third-party data sharing.

No API key is needed. Specola uses the Claude Code CLI, which manages authentication through its own session. Just make sure claude is installed and in your PATH.

Specola degrades gracefully. If the Claude CLI is unreachable or times out, the app still generates a DOCX document containing the raw feed content, with a red warning banner reading "Analisi non disponibile." You always get a briefing — the AI analysis is an enhancement, not a hard dependency.

The five-section structure (Da sapere oggi, Richiede attenzione, Per area tematica, Da leggere con calma, Spunti) is the default format, optimized for professional knowledge workers. Your professional profile influences the content and prioritization within each section.

Specola has been tested with 200+ feeds and handles them comfortably thanks to its concurrent 20-thread fetching pool. Failed or slow feeds are skipped without blocking the rest. The practical limit depends on your network speed and the Claude context window.

Specola listens for NSWorkspace wake notifications. When your Mac wakes up, it checks whether a scheduled briefing was missed and runs it immediately. No cron jobs, no launchd agents — it's all handled internally.
